# 数仓建表 AI 助手 PRD

## 1. 产品概述
数仓建表 AI 助手是一款面向数据仓库开发人员的 DDL 生成工具。用户输入业务描述、词根库、开发规范和数据库类型后，系统自动生成符合命名规则和注释规范的建表 SQL，并支持批量 Excel 导入、规范管理、历史记录、数据库连接测试和历史 SQL 执行。

## 2. 目标
- 提升数据建表效率，减少人工编写 DDL 的重复劳动。
- 统一表名、字段名、字段类型和注释规范。
- 通过词根库复用和规范库约束，提高命名一致性。
- 支持单表生成、批量生成、历史回溯和落库执行。

## 3. 目标用户
- 数据仓库开发人员
- 数仓建模人员
- 需要批量整理表结构的实施人员

## 4. 产品范围
### 4.1 当前包含
- 单表 DDL 生成
- 批量 Excel 建表生成
- 词根库管理
- 开发规范管理
- 自定义提示词管理
- LLM 配置与连接测试
- 数据库连接管理
- DDL 历史记录管理
- 历史 SQL 执行与下载
- 生成进度展示与任务取消

### 4.2 明确不包含
- 直接在单表生成流程里执行数据库写入
- 多项目/租户隔离
- 复杂工作流编排
- 业务建模辅助以外的通用 BI 能力

## 5. 核心能力
### 5.1 单表生成
- 输入业务描述、数据库类型、词根优先级、是否启用校验、自定义提示词。
- 调用 LLM 生成表名和字段名。
- 按开发规范补充表注释、字段注释和类型映射。
- 输出 DDL 和新词根结果。

### 5.2 批量生成
- 通过 Excel 导入多表结构。
- 采用字段级处理流程，先分词、再匹配历史词根、再由 LLM 补全未命中部分。
- 支持并发生成、进度查询和任务取消。
- 支持统一校验和统一修正。

### 5.3 规范与词根管理
- 维护自定义规范与内置默认规范。
- 维护全局词根库，支持新增、删除、编辑、搜索、保存。
- 支持词根优先级切换：优先全称 / 优先缩写。

### 5.4 历史与执行
- 保存每次生成记录。
- 支持下载历史 DDL。
- 支持把历史 DDL 绑定到已保存数据库连接后直接执行。
- 执行结果会回写到历史记录中。

## 6. 关键流程
### 6.1 单表生成流程
1. 用户填写 LLM 配置、业务描述和生成参数。
2. 后端加载词根、规范和自定义提示词。
3. 拼装提示词并请求 LLM。
4. 后端进行规范校验、必要时修正。
5. 返回 DDL、新词根和校验结果。

### 6.2 批量生成流程
1. 用户上传 Excel。
2. 后端解析表结构并构建字段映射。
3. 字段级处理分三层：完全命中、部分命中、完全未命中。
4. 组装各表 DDL。
5. 可选执行统一校验与修正。
6. 返回批量结果、错误项、耗时和新词根。

### 6.3 历史执行流程
1. 用户在历史记录中选择一条 DDL。
2. 选择数据库连接。
3. 后端校验脚本类型与连接类型是否一致。
4. 通过原生数据库驱动执行 SQL。
5. 写回执行状态与执行信息。

## 7. 页面模块
- 基础配置：LLM 配置、连接测试、数据库连接管理。
- 词根输入：手动录入、文件导入、模板下载。
- 自定义提示词：编辑、保存、重置。
- 单一建表：需求输入、DDL 输出、校验提示、新词根展示。
- 批量建表：文件导入、进度展示、结果展示、错误展示。
- 开发规范：规范列表、启用/禁用、编辑、删除、重置、上传。
- 建表历史：历史列表、下载、执行、删除。

## 9. 数据与存储
- `word_roots.json`：词根库
- `custom_prompt.txt`：自定义提示词
- `dev_standards.json` 与 `standards/`：规范库
- `ddl_history.json`：生成历史和执行历史
- `db_connections.json`：数据库连接信息
- `db_secret.key`：数据库密码加密相关密钥
- `logs/`：运行日志

## 10. 约束与规则
- 单表生成以复制 DDL 为主，不在该流程里直接落库。
- 历史 SQL 执行只允许在历史记录流程中进行。
- 字段命名必须服从词根库和当前优先级模式。
- 表名必须符合业务域前缀和数据库类型约束。
- 开发规范优先级高于单次提示词。

## 11. 成功标准
- 用户能在单表场景稳定生成可用 DDL。
- 批量场景能稳定处理 Excel，多表生成结果可追踪。
- 新词根能被记录并复用。
- 历史记录支持回看、下载和执行。
- 规范切换后，生成结果仍保持一致性。

## 12. 主要风险
- 词根库污染会直接影响命名一致性。
- 规范文件与内置默认文件不同步会导致预期不一致。
- 批量任务的校验和修正链路较长，需关注超时与中断。
- 历史执行与连接类型不匹配会导致脚本执行失败。

## 13. 当前结论
这个项目已经不是“只生成一段 SQL”的工具，而是一个围绕数仓 DDL 的完整工作台：生成、校验、规范、词根沉淀、历史执行都已经纳入主流程。后续任何功能设计都应优先遵守“单表生成只产出，历史流程才执行”的边界。
