# 数仓AI助手 - 部署指南 v2.0

## 快速开始（3分钟上手）

### 方式一：已安装 Python 环境
1. 双击运行 `SmartLauncher.bat`
2. 等待自动检测并安装依赖
3. 浏览器自动打开，访问 http://localhost:5173

### 方式二：首次使用（需要配置环境）
1. 双击运行 `EnvSetup.bat`
2. 按照提示安装 Python（如未安装）
3. 双击运行 `SmartLauncher.bat`

## 端口配置

### 默认端口
| 服务 | 端口 | 访问地址 |
|------|------|---------|
| **前端** | 5173 | http://localhost:5173 |
| **后端** | 8000 | http://localhost:8000 |

### 如何修改端口

如果需要修改端口，编辑 `SmartLauncher.bat` 文件顶部的配置：

```batch
REM ================================================
REM CONFIGURATION - Change these values as needed
REM ================================================
set "BACKEND_PORT=8000"
set "FRONTEND_PORT=5173"
REM ================================================
```

**注意**：修改端口后需要重启脚本

## 一键打包（生成独立运行版本）

如果您需要将程序分享给没有 Python 环境的用户：

1. 确保已安装 Python 环境
2. 双击运行 `PackageApp.bat`
3. 等待打包完成（约3-5分钟）
4. 打包完成后，进入 `dist\DW_Backend_Service` 目录
5. 将整个目录分享给用户即可

**打包后的文件使用方式：**
- 运行 `DW_AI_Backend.exe`
- 浏览器访问 http://localhost:8000

**优点：对方完全不需要安装任何环境！**

## 常见问题

### Q1: 提示"Python environment NOT detected"

**原因**：系统中未安装 Python 或未添加到 PATH

**解决方案**：
1. 下载 Python：https://www.python.org/downloads/
2. 安装时勾选 "Add Python to PATH"
3. 重新运行 `SmartLauncher.bat`

### Q2: 提示"Port is already in use"

**原因**：端口已被其他程序占用

**解决方案**：
1. 脚本会自动尝试关闭占用端口的进程
2. 或手动关闭占用端口的程序
3. 或修改脚本中的端口配置

### Q3: 依赖安装失败

**解决方案**：
1. 检查网络连接
2. 脚本已自动使用国内镜像加速
3. 手动安装（可选）：
   ```bash
   pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
   ```

### Q4: 浏览器无法访问

**检查项**：
1. 服务是否启动成功（检查命令行窗口）
2. 端口是否正确（应为 5173）
3. 防火墙是否阻止

### Q5: 打包失败

**可能原因**：
1. PyInstaller 未安装 - 脚本会自动安装
2. 缺少依赖 - 确保 requirements.txt 完整
3. 路径问题 - 确保在项目根目录运行脚本

## 技术支持检查清单

如遇问题，请按顺序检查：

1. **Python 是否正确安装**
   ```bash
   python --version
   ```

2. **依赖是否完整**
   ```bash
   pip list
   ```

3. **端口是否可用**
   ```bash
   netstat -ano | findstr 5173
   netstat -ano | findstr 8000
   ```

4. **防火墙设置**
   - 允许 Python 和 Node.js 通过防火墙
   - 或直接关闭防火墙测试
